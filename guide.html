구글 시트
연동 가이드

운영진이 구글 스프레드시트 한 장만 관리하면 사이트가 자동으로 동기화되도록 만드는 방법. 일정 변경 / 참가자 추가 / 공지 업데이트 모두 시트에서만. 개발자 도움 없이도 콘텐츠 운영 가능.

목차

  1. 구글 시트 만들기 (10분)
  2. 시트 양식 — 6개 탭 스펙
  3. 웹에 게시 + URL 복사
  4. data.js 코드 교체
  5. 운영 팁 · 캐싱 · 보안

STEP 01구글 시트 만들기

  1. 구글 드라이브에서 새 스프레드시트 생성 → 이름: MACAU RUNFLUENCER 2026
  2. 하단에 6개 탭 만들기 — SCHEDULE · PEOPLE · PERSONAL · PLACES · ANNOUNCEMENTS · CHECKLIST
  3. 각 탭의 1행은 고정 헤더 (정확한 컬럼명 사용 · 다음 섹션 참조)
  4. 2행부터 데이터 입력
샘플 시트 템플릿 — 아래 5개 탭의 컬럼 명세 그대로 복사해서 만들거나, 샘플 CSV 다운로드 후 시트에 import 하세요.

STEP 02시트 양식 — 6개 탭 스펙

▸ SCHEDULE 탭

일정 항목 (각 행 = 일정 1개). DAY는 1/2/3, 같은 날짜 일정은 모두 같은 DAY로.

SCHEDULE필수 탭
컬럼명타입필수설명
day숫자●1, 2, 3
dateYYYY-MM-DD●2026-06-26 (텍스트로 입력 권장)
day_label텍스트○FRI / SAT / SUN
day_title텍스트○일자 제목 (예: COTAI CITY RUN)
timeHH:MM●17:00 (24시간 텍스트)
endHH:MM○종료 시각 (없으면 비움)
title텍스트●COTAI CITY RUN
sub텍스트○부제 / 카테고리
icon이모지○🏃 / ✈️ / 🍽️ 등
desc텍스트○모달에 표시될 상세 설명
location텍스트○장소명

rsvpTRUE/FALSE○참석 체크 받는 자유시간 활동
예시 행 → 1 | 2026-06-26 | FRI | COTAI CITY RUN | 17:00 | 18:30 | COTAI CITY RUN | Hero session · 6.5km | 🏃 | 코타이 야경 라이트 런 6.5km | Cotai Strip | TRUE | FALSE

▸ PERSONAL 탭 ★ 핵심

각자 자유시간에 뭐 할지를 한 행씩 적는 탭. 같은 activity를 적은 사람들이 자동으로 그룹핑돼서 사이트에 "같이 가는 사람" 으로 묶여 표시됨. 참가자가 직접 적거나 운영진이 취합해서 정리해도 됨.

PERSONAL필수 탭
컬럼명타입필수설명
name텍스트●PEOPLE 탭의 name과 정확히 일치해야 자동 매칭
slot텍스트●d1-night / d2-pm / d3-am 중 하나
activity텍스트●활동명. 같은 글자 그대로 = 같은 그룹 (오타 주의)
note텍스트○"21:30 로비" 같은 메모. 비워둬도 OK
slot ID 의미 → d1-night → DAY 1 (06.26 금) · 21:30 ~ 자유시간 (디너 후 야간) d2-pm → DAY 2 (06.27 토) · 11:30 ~ 자유시간 (트라이 이후 오후·저녁) d3-am → DAY 3 (06.28 일) · 09:00 ~ 자유시간 (헤리티지 런 이후)
예시 행 → 박수현 | d1-night | 베니션 야경 산책 | 21:30 로비 철민 | d1-night | 베니션 야경 산책 | 정다영 | d1-night | 카지노 투어 · 갤럭시 | ID 필수 체리핏 | d1-night | 호텔 라운지 · 휴식 | → 박수현 + 철민 = 같은 활동이라 자동으로 한 그룹 (2명) → 정다영, 체리핏은 각자 다른 활동 그룹
운영 팁: 활동 종류를 미리 5-7개 정해두고 드롭다운으로 선택하게 하면 오타 방지에 좋아요. 구글 시트 메뉴: 데이터 → 데이터 확인 → 목록에서 선택으로 드롭다운 설정 가능.

STEP 03웹에 게시 + URL 복사

  1. 시트 메뉴 → 파일 → 공유 → 웹에 게시
  2. 게시 대상: 전체 문서, 형식: 웹페이지 (.html) 선택 후 게시
  3. 접근 권한 확인 → 일반 액세스를 "링크가 있는 모든 사용자"로 (보기 권한)
  4. 각 탭의 데이터를 JSON으로 받으려면 gviz 엔드포인트 사용 — 시트 ID 복사:
COPY// 시트 URL에서 ID만 추출 → https://docs.google.com/spreadsheets/d/[SHEET_ID]/edit
// SHEET_ID 부분만 복사. 길고 랜덤한 문자열.

// 각 탭 JSON URL 형식:
https://docs.google.com/spreadsheets/d/SHEET_ID/gviz/tq
  ?tqx=out:json
  &sheet=SCHEDULE      // 탭 이름 (정확히)
  &headers=1            // 1행 헤더 사용
주의: 응답은 일반 JSON이 아니라 google.visualization.Query.setResponse({...})로 감싸진 형태로 옴. 파싱할 때 앞 47글자 잘라내고 끝의 ); 떼야 함. 아래 코드에서 자동 처리.

STEP 04data.js 코드 교체

현재 data.js의 하드코딩된 데이터를 시트에서 fetch 하는 코드로 교체. 아래 코드 전체 복사 → data.js 맨 아래에 추가:

COPY// ─── Google Sheets 연동 ───────────────────────────────
// 시트 ID만 바꾸면 끝. 사이트 로드 시 자동으로 시트에서 데이터 받아 덮어씀.
const SHEET_ID = "YOUR_SHEET_ID_HERE";

async function fetchTab(name) {
  const url = `https://docs.google.com/spreadsheets/d/${SHEET_ID}/gviz/tq?tqx=out:json&sheet=${name}&headers=1`;
  const res = await fetch(url);
  const txt = await res.text();
  // google.visualization.Query.setResponse({...})  → 가운데 JSON만 추출
  const json = JSON.parse(txt.match(/\(([\s\S]+)\);?$/)[1]);
  const cols = json.table.cols.map(c => (c.label || c.id).toLowerCase());
  return json.table.rows.map(r => {
    const obj = {};
    r.c.forEach((cell, i) => obj[cols[i]] = cell ? cell.v : null);
    return obj;
  });
}

async function loadFromSheet() {
  try {
    const [sched, ppl, plc, ann, chk, pers] = await Promise.all([
      fetchTab("SCHEDULE"),    fetchTab("PEOPLE"),
      fetchTab("PLACES"),      fetchTab("ANNOUNCEMENTS"),
      fetchTab("CHECKLIST"),   fetchTab("PERSONAL"),
    ]);

    // SCHEDULE: group items by day
    const byDay = {};
    sched.forEach(r => {
      const d = r.day;
      if (!byDay[d]) byDay[d] = {
        day: d, date: r.date, label: r.day_label,
        title: r.day_title, items: []
      };
      byDay[d].items.push({
        time: r.time, end: r.end, title: r.title, sub: r.sub,
        icon: r.icon, desc: r.desc, location: r.location,
        hero: r.hero === true, rsvp: r.rsvp === true,
      });
    });
    window.SCHEDULE = Object.values(byDay).sort((a,b) => a.day - b.day);

    window.PEOPLE = ppl.map(r => ({
      bib: r.bib, name: r.name, ig: r.ig, role: r.role,
      lead: r.lead === true, photo: r.photo,
    }));

    window.PLACES = plc.map(r => ({
      type: r.type, name: r.name, desc: r.desc, img: r.img,
      lat: r.lat, lng: r.lng,
      stats: [[r.stat1_v, r.stat1_l], [r.stat2_v, r.stat2_l], [r.stat3_v, r.stat3_l]]
        .filter(s => s[0] != null),
    }));

    window.ANNOUNCEMENTS = ann.map(r => ({
      date: r.date, title: r.title, desc: r.desc, pin: r.pin === true
    }));

    // CHECKLIST: group items by category
    const byCat = {};
    chk.forEach(r => {
      if (!byCat[r.category]) byCat[r.category] = [];
      byCat[r.category].push(r.item);
    });
    window.CHECKLIST = byCat;

    // PERSONAL: 그대로 배열로 (사이트에서 slot + activity로 그룹핑)
    window.PERSONAL = pers.map(r => ({
      name: r.name, slot: r.slot, activity: r.activity, note: r.note || "",
    }));

    // 페이지 다시 렌더 (React 앱)
    window.dispatchEvent(new Event("sheet-loaded"));
  } catch (e) {
    console.warn("시트 로드 실패. 하드코딩 데이터로 fallback.", e);
  }
}

loadFromSheet();

STEP 05운영 팁 · 캐싱 · 보안

캐싱 (시트 부하 방지)

  • 위 코드는 페이지 로드마다 시트 호출 — 트래픽 적으면 OK
  • 참가자가 자주 새로고침 → localStorage에 1분 캐싱 권장:
COPYconst CACHE_MS = 60 * 1000; // 1분
async function fetchTab(name) {
  const key = `sheet-${name}`;
  const cached = JSON.parse(localStorage.getItem(key) || "null");
  if (cached && Date.now() - cached.t < CACHE_MS) return cached.d;

보안 · 권한

  • 시트는 읽기 전용 공개 — 누가 사이트 보더라도 시트 직접 수정 불가
  • 운영진만 시트 편집 권한 (구글 계정으로 별도 공유)
  • 민감 정보 (전화번호, 비행기 좌석 등)는 시트에 올리지 말 것 — 공개 URL이라 누구나 fetch 가능

자주 묻는 실수

  • 탭 이름이 정확히 SCHEDULE / PEOPLE 등 대문자 일치해야 함
  • HH:MM 입력 시 구글 시트가 자동으로 "오후 5:00" 같은 형식으로 바꿔버림 → 셀 형식을 일반 텍스트로 미리 설정
  • 날짜도 마찬가지로 텍스트 형식 권장 (2026-06-26)
  • TRUE/FALSE는 시트의 체크박스 기능 사용 추천 (삽입 → 체크박스)
